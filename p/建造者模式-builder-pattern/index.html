建造者模式 (Builder Pattern)

分类: 创建型模式 (Creational)

要解决的问题

假设你要组装一台电脑，有以下选择：

1
2
3
4
5

CPU:    i5-13400 / i7-14700K / Ryzen 9 7950X
GPU:    集成显卡 / RTX 4080 / RTX 4090
内存:   16GB DDR4 / 32GB DDR5 / 64GB DDR5
硬盘:   512GB SSD / 2TB NVMe / 4TB NVMe
系统:   Windows 11 / Arch Linux / macOS

如果用构造函数：

1

Computer(string cpu, string gpu, string mem, string storage, string os);

问题是：

  • 参数太多，调用时不知道每个位置对应什么
  • 很多配置有默认值，但仍需传参
  • 同样的构建步骤（设置 CPU → GPU → 内存 → 硬盘 → 系统）可能需要产出不同的配置

建造者模式将"构建过程"与"内部表示"分离，让同样的构建步骤产出不同的产品。

                    ┌────────┴────────┐
                    ▼                 ▼
         GamingBuilder          OfficeBuilder
          (游戏电脑)              (办公电脑)
                    │                 │
                    └───────┬─────────┘
                            ▼
                       Computer (产品)

  客户端调用流程:

  方式1: Director → Builder.setXxx() × N → build() → Product
  方式2: Builder.setXxx() × N → build() → Product  (免Director)

三个核心角色：

角色类职责
Product (产品)Computer被构建的复杂对象
Builder (建造者接口)ComputerBuilder定义构建步骤的抽象接口
ConcreteBuilder (具体建造者)GamingComputerBuilder / OfficeComputerBuilder实现具体的构建逻辑
Director (指导者)Director控制构建步骤的顺序（可选）

与工厂模式的区别

1
2
3

工厂方法     → 选哪个产品          (createAnimal("dog") → Dog)
抽象工厂     → 选哪套产品族        (createWindowsGUI() → WinButton + WinText)
建造者模式   → 怎么一步步装出来    (builder.setCpu().setGpu().build() → Computer)

关键代码

Builder 接口

每个 setter 返回引用以支持链式调用：

class ComputerBuilder {
public:
    virtual ~ComputerBuilder() = default;
    virtual ComputerBuilder& setCpu(const std::string& cpu) = 0;
    virtual ComputerBuilder& setGpu(const std::string& gpu) = 0;
    virtual ComputerBuilder& setMemory(const std::string& memory) = 0;
    virtual ComputerBuilder& setStorage(const std::string& storage) = 0;
    virtual ComputerBuilder& setOs(const std::string& os) = 0;
    virtual std::unique_ptr<Computer> build() = 0;
};

具体建造者 — 办公电脑

OfficeBuilder 在 setGpu() 和 setOs() 中做了策略覆盖：

class OfficeComputerBuilder : public ComputerBuilder {
    // ...
    ComputerBuilder& setGpu(const std::string& gpu) override {
        // 办公电脑强制使用集成显卡
        computer_->setGpu("Integrated (forced by office policy)");
        return *this;
    }
    ComputerBuilder& setOs(const std::string& os) override {
        computer_->setOs(os + " (Enterprise Edition)");
        return *this;
    }
};

// 方式1: 通过 Director（预定义配置）
Director director;
GamingComputerBuilder builder;
director.setBuilder(&builder);
auto pc = director.buildStandardGamingPC();

// 方式2: 链式调用（更灵活）
auto pc = GamingComputerBuilder()
    .setCpu("AMD Ryzen 9 7950X")
    .setGpu("NVIDIA RTX 4090")
    .setMemory("64GB DDR5")
    .setStorage("4TB NVMe SSD")
    .setOs("Arch Linux")
    .build();
