Linux 5.15.78 内核源码架构分析

一、整体架构分层

Linux 内核采用宏内核(Monolithic)架构，所有核心服务运行在同一特权级(Ring 0)的同一地址空间，通过函数调用而非 IPC 通信，兼具高性能和模块可加载的灵活性。

drivers/设备驱动字符/块/网络/平台设备、总线(PCI/USB/I2C)、Android Binder

3.4 Initcall 层次体系

do_initcalls() 按段地址顺序执行 8 个级别的初始化函数：

级别宏段名用途典型注册者
0pure_initcall.initcall0纯变量初始化，不依赖任何子系统少数底层设施
1core_initcall.initcall1内核核心依赖内存、中断、基础设施
2postcore_initcall.initcall2依赖 core 的后续逻辑kobject、irq_domain
3arch_initcall.initcall3体系结构特定初始化IOMMU、ACPI、平台
4subsys_initcall.initcall4大型子系统框架PCI、USB、I2C 总线
5fs_initcall.initcall5文件系统/协议族注册ext4、inet_init()、proc
6device_initcall.initcall6设备驱动注册（module_init() 默认）各种驱动程序
7late_initcall.initcall7启动尾声，依赖面最广调优、可选特性

另有 early_initcall 在 SMP 之前由 do_pre_smp_initcalls() 单独执行。

struct task_struct {
    // ─── 调度相关 ───
    __state                     运行状态 (RUNNING/INTERRUPTIBLE/...)
    prio / static_prio          优先级
    sched_class                 调度类指针 (stop/dl/rt/fair/idle)
    se (sched_entity)           CFS 调度实体 (含 vruntime / rb_node)
    rt (sched_rt_entity)        RT 调度实体
    on_rq                       是否在运行队列

    // ─── 内存 ───
    mm / active_mm              用户地址空间 / 活跃mm(内核线程借用)

    // ─── 文件系统 ───
    fs (fs_struct)              当前目录/根目录/umask
    files (files_struct)        打开文件表

    // ─── 信号 ───
    signal (signal_struct)      线程组共享信号状态
    sighand                     信号处理函数表
    pending                     私有待处理信号队列

    // ─── 标识 ───
    pid                         线程 ID
    tgid                        线程组 ID（用户态看到的"进程 ID"）
    stack                       内核栈指针
};

per-CPU 运行队列 (struct rq)
  └── cfs_rq
        └── 红黑树 (按 vruntime 排序)
              ├── task A (vruntime=100)
              ├── task B (vruntime=120)  ← 最左节点 = 下一个运行

调度循环:
  schedule() → __schedule()
    → pick_next_task()
      → pick_next_task_fair()
        → pick_next_entity()    取 vruntime 最小的实体
    → context_switch()
        ├── switch_mm()         切换页表 (CR3/TTBR0)
        └── switch_to()         切换寄存器和栈（汇编）

每次运行时 update_curr():
  vruntime += delta_exec × (NICE_0_WEIGHT / task_weight)
  → 高权重任务 vruntime 增长慢，获得更多 CPU 时间

六、三个特殊进程

PID名称创建方式职责
0swapper/idle静态定义 init_task（init/init_task.c），非动态创建执行 start_kernel() 全部初始化，然后永久进入 cpu_startup_entry() 的 idle 循环
1initrest_init() 中 kernel_thread(kernel_init)执行 do_initcalls()，然后 kernel_execve("/sbin/init") 变身为用户态 init 进程，是所有用户进程的祖先
2kthreaddrest_init() 中 kernel_thread(kthreadd)内核线程守护进程，所有内核线程（kworker/ksoftirqd/kswapd 等）的创建都由它代理完成
